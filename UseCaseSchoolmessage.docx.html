Use Case Name

School Message

Author: Quentin

Use Case Id

Date: 25/11/2015

Priority

Source

Requirements

Primary Business Actor

Student

Other Participating Actors

Description

As a student I can post an new message in my School Group

Preconditions

The Student must to be registered.

Trigger

Typical Scenario

Actor Action

System Response

Step 1: The student click on the “My School” tab.

Step 3: The student write the message inside the chatBox and send it.

Step 2: The  system display the School’s wall interface.

Step 4: The  system display the new message School’s wall interface.

Step 5 : The system send a notification to each school’s students.

Alternate Scenarios

Actor Action

System Response

Required field not entered

Step 1:The student send a message without writing anything

Step 2:The system doesn’t do anything.

Conclusions

The message has successfully been posted.

Post conditions

Business Rules

You have to be friend with the student you want to chat.

Implementation Constraints
