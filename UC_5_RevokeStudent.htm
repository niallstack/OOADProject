Student

 

Use Case Name

Revoke all permissions for a student

Author: Alex

Use Case Id

05

Date: 25-11-15

Priority

 

Post conditions

The revoked is student is tagged as inactive on the system

Primary Business Actor

School Office Administrator (SOA)

Other Participating Actors

N/A

Description

This function enables the admin to delete a student from the system.

Preconditions

The student must already be registered in the system and the School Office administrator is logged on.

Typical Scenario

Actor Action

 

1: The SOA selects to revoke a student

 

2: The system displays the revoke student blank form, UI.

 

3: The SOA enters the students

 

4: The system determines if the ID exists and displays the students information.

 

5: The SOA selects the submit button

 

6: The system displays a message asking the SOA to confirm with the yes and no dialog.

 

7: The SOA confirms

 

8: The students is then revoked all permissions on the system

 

9: The system displays a confirmation message and tags the student as inactive on the system

 

10: The system returns the SOA back to the home page.

Alternate Scenarios

Actor Action

Student ID Does not exist

4.a: Student is not registered with the system.

 

5.a: The system displays an appropriate error message

 

 

The admin clicks No in the confirmation dialog.

6.a: The system returns the SOA back to the Home page of the system.

Conclusions

A student has been revoked all permissions from the system.

Business Rules

Student must be already registered

Implementation Constraints
