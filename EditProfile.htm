Description

This function enables the actors to edit students information.

Preconditions

Trigger

A students file is requested in order to edit current information.

Typical Scenario

Actor Action

System Response

 

Step 1: The student invokes the amend profile function.

 

Step 3: The student enters new values in either of these fields:

         Surname

         Forename

         DOB

         Class

         School

         Profile picture

 

Step 4: The student confirms the new information by clicking the Amend function.

Step 2: The system determines the student ID and the amend profile UI is displayed.

 

 

 

 

 

 

 

 

 

Step 5: The system validates the data entered:

         DOB must be in dd/mm/yyyy format

 

 

Step 6: The new student details are saved in the students file.

 

Step 7: The system displays a confirmation message

 

Step 8: The system returns the student back to the view profile page.

Alternate Scenarios

Actor Action

System Response

DOB invalid

 

 

 

Step 4: DOB is in the future

 

Step 5: The system displays an appropriate error message

Conclusions

A students file has been amended.
